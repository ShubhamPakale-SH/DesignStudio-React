================================================================
           MESSAGE QUEUE ARCHITECTURE
================================================================

WHAT IS A MESSAGE QUEUE?
--------------------------
A Message Queue is a form of asynchronous service-to-service
communication used in distributed systems. Messages are stored
in a queue until they are processed and deleted. It decouples
the producer (sender) from the consumer (receiver), allowing
them to operate independently.


CORE COMPONENTS
--------------------------
1. PRODUCER
   - The entity that creates and sends messages to the queue.
   - Also called "Publisher" in pub/sub models.

2. MESSAGE QUEUE (BROKER)
   - A temporary storage buffer that holds messages.
   - Ensures messages are delivered even if the consumer is
     temporarily unavailable.
   - Examples: RabbitMQ, Apache Kafka, AWS SQS, ActiveMQ.

3. CONSUMER
   - The entity that reads and processes messages from the queue.
   - Also called "Subscriber" in pub/sub models.
   - Can be one or many consumers (competing consumers pattern).

4. MESSAGE
   - The data unit passed between producer and consumer.
   - Can be plain text, JSON, XML, binary data, etc.


HOW IT WORKS (FLOW)
--------------------------
  [Producer] --> [Exchange] --> [Queue] --> [Consumer]

  Step 1: Producer sends a message to an Exchange.
  Step 2: Exchange routes it to one or more Queues.
  Step 3: Queue stores the message persistently.
  Step 4: Consumer receives and processes the message.
  Step 5: Consumer sends ACK; broker removes the message.

           RABBITMQ ARCHITECTURE
================================================================

WHAT IS RABBITMQ?
--------------------------
RabbitMQ is an open-source message broker that implements the
Advanced Message Queuing Protocol (AMQP). It acts as a middleman
for messages, allowing producers and consumers to communicate
reliably and asynchronously.


RABBITMQ KEY COMPONENTS
--------------------------
1. CONNECTION
   - A TCP connection between your app and RabbitMQ broker.

2. CHANNEL
   - A virtual connection inside a TCP connection.
   - Most operations (publish, consume) happen over channels.

3. EXCHANGE
   - Receives messages from producers and routes them to queues.
   - Types:
     a) Direct  - Routes to queue with exact routing key match.
     b) Fanout  - Broadcasts to all bound queues.
     c) Topic   - Routes based on pattern matching (wildcards).
     d) Headers - Routes based on message header attributes.

4. QUEUE
   - Stores messages until consumed.
   - Properties: durable, exclusive, auto-delete.

5. BINDING
   - A link between an exchange and a queue with a routing key.

6. ROUTING KEY
   - An attribute the exchange uses to route messages.

7. VIRTUAL HOST (vHost)
   - Logical separation of resources inside one RabbitMQ broker.


RABBITMQ MESSAGE FLOW
--------------------------
  Producer
     |
     v
  [Exchange]  ---- binding (routing key) ---->  [Queue A]
                                           ---->  [Queue B]
                                                     |
                                                     v
                                                 Consumer


EXCHANGE TYPES EXPLAINED
--------------------------
Direct Exchange:
  Producer sends "order.created" --> Queue bound to "order.created"

Fanout Exchange:
  Producer sends message --> ALL bound queues receive it
  (used for broadcast / notifications)

Topic Exchange:
  Producer sends "order.created.us" -->
  Queue bound to "order.#" receives it  (# = any words)
  Queue bound to "*.created.*" also receives it

Headers Exchange:
  Routing based on message headers instead of routing key.


STEPS TO SET UP RABBITMQ WITH JAVA
--------------------------
Step 1: Install RabbitMQ
  - Download from: https://www.rabbitmq.com/download.html
  - Or run with Docker:
      docker run -d --name rabbitmq \
        -p 5672:5672 -p 15672:15672 \
        rabbitmq:3-management
  - Access management UI at: http://localhost:15672
    (default user: guest / password: guest)

Step 2: Add RabbitMQ Java Client Dependency
  Maven (pom.xml):
    <dependency>
      <groupId>com.rabbitmq</groupId>
      <artifactId>amqp-client</artifactId>
      <version>5.20.0</version>
    </dependency>

  Gradle (build.gradle):
    implementation 'com.rabbitmq:amqp-client:5.20.0'

Step 3: Create a ConnectionFactory
  ConnectionFactory factory = new ConnectionFactory();
  factory.setHost("localhost");
  factory.setPort(5672);
  factory.setUsername("guest");
  factory.setPassword("guest");

Step 4: Create Connection and Channel
  Connection connection = factory.newConnection();
  Channel channel = connection.createChannel();

Step 5: Declare Exchange and Queue
  channel.exchangeDeclare("order_exchange", "direct", true);
  channel.queueDeclare("order_queue", true, false, false, null);
  channel.queueBind("order_queue", "order_exchange", "order.created");

Step 6: Publish a Message (Producer)
  String message = "Order #1001 created";
  channel.basicPublish("order_exchange", "order.created",
      MessageProperties.PERSISTENT_TEXT_PLAIN,
      message.getBytes(StandardCharsets.UTF_8));

Step 7: Consume Messages (Consumer)
  channel.basicConsume("order_queue", false, (tag, delivery) -> {
      String msg = new String(delivery.getBody(), StandardCharsets.UTF_8);
      System.out.println("Received: " + msg);
      channel.basicAck(delivery.getEnvelope().getDeliveryTag(), false);
  }, tag -> {});

Step 8: Enable Fair Dispatch (optional but recommended)
  channel.basicQos(1);  // Process one message at a time per consumer


KEY RABBITMQ CONCEPTS
--------------------------
- Durable Queue    : Survives broker restart.
- Persistent Msg   : Stored to disk, survives restart.
- ACK              : Consumer confirms successful processing.
- NACK / Reject    : Consumer signals failure; message retried.
- Prefetch (QoS)   : Limits unacknowledged messages per consumer.
- Dead Letter Exch : Failed messages routed to a DLX for analysis.
- TTL              : Message or queue expiry time.
- Publisher Confirm: Broker confirms message was received safely.


ADVANTAGES OF RABBITMQ
--------------------------
- Supports multiple protocols (AMQP, MQTT, STOMP)
- Flexible routing via exchange types
- High availability with clustering and mirroring
- Built-in management UI
- Strong community and wide language support
- Message persistence and durability guarantees


COMMON USE CASES
--------------------------
- Microservices event-driven communication
- Order processing pipelines
- Email / SMS notification delivery
- Task queues for background jobs
- Log aggregation systems
- IoT device data ingestion
